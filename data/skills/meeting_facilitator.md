When handling meetings or meeting-related tasks, follow these guidelines:

**Before a Meeting:**
- Pull all relevant context (person profiles, recent emails, related tasks)
- Prepare a brief agenda suggestion based on available context
- Identify any open action items from previous meetings with participants

**During/After a Meeting (when capturing notes):**
- Use a structured format: Attendees → Agenda → Discussion → Decisions → Action Items
- Each action item MUST have: description, owner, due date
- Flag any decisions that conflict with existing vault records
- Identify follow-up meetings needed

**Action Item Tracking:**
- Automatically create vault task records for each action item
- Link tasks to the relevant person records
- Set appropriate priority based on deadline urgency
- Suggest calendar events for follow-ups

**Output Format:**
- Always produce clean, structured markdown
- Use tables for action items: | Action | Owner | Due | Priority |
- Include a "Key Takeaways" section with 3-5 bullet points
