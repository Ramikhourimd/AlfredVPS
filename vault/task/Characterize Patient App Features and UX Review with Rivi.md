# Characterize Patient App Features and UX Review with Rivi

## Description

Before the patient-facing mobile app can be rolled out broadly to all patients (as Dekkel is pushing for), the app needs a comprehensive UX characterization. Rami proposed involving Rivi Idelman for this evaluation.

## Acceptance Criteria

- [ ] Comprehensive review of what patients currently see in the app
- [ ] Inventory of available features (download summaries, schedule appointments, etc.)
- [ ] Identify what complaint handling paths exist (or are missing)
- [ ] Rivi Idelman completes UX evaluation
- [ ] Report of findings shared with product team

## Context

Raised during the 2025-11-10 product meeting. Rami expressed concern that launching to all patients without understanding the current state would cause avoidable complaints and erode patient trust.
