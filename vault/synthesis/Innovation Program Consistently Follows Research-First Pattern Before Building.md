# Innovation Program Consistently Follows Research-First Pattern Before Building

## Insight

Across every major initiative in the Telia'z Innovation Program, a consistent organizational pattern emerges: the team always establishes a research/framework phase before any building or execution begins. This is not accidental — it is an enforced methodology driven by Rami's leadership style and formalized through Rivi's role definition as researcher/concept designer (not executor).

## Evidence

Four independent workstreams all follow the same pattern:

1. **AI Model Evaluation** (Jan 2025): Built rubric-based evaluation framework and scoring methodology before comparing models. Onboarded external advisor for methodology rigor before running evaluations.

2. **Staff Experience Research** (Feb 2026): Rami directed Rivi to treat qualitative findings as academic research — build a structured survey instrument with measurable variables before acting on any burnout intervention.

3. **Next-Generation Clinical System Prototype** (Feb 2026): Survey all existing clinical systems in the market and build composite model before designing anything new. Research first, prototype second.

4. **Pre-Treatment Patient Video** (Feb 2026): Rivi explicitly redirected from producing video content to writing a concept document first — define the problem, research basis, and approach before any production work.

## Implications

- New innovation initiatives should expect a mandatory research/framework phase before execution
- Team members who skip to building will be redirected (as happened with Rivi multiple times)
- The top-down gated concept design framework formalizes this into process: problem definition → research → concept → approval gate → execution handoff
- This pattern trades speed for rigor — it prevents wasted execution effort but may slow time-to-delivery

## Applicability

Applies to all Telia'z Innovation Program workstreams. Any new initiative proposed to Rami should include an explicit research phase in its plan.
