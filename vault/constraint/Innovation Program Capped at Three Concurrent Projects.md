# Innovation Program Capped at Three Concurrent Projects

## Constraint

The Telia'z Innovation Program enforces a work-in-progress (WIP) limit of maximum 3 concurrent projects at any time. This cap governs the team's parallel capacity — Rivi, Rami, and team members cannot take on a fourth initiative until one of the active three is completed or paused.

## Source

Defined as part of the Innovation Program's top-down, template-driven concept design operating model. The WIP limit complements the gated approval process where Rami gates each project before execution.

## Implications

- New innovation requests must queue when three projects are active
- Forces explicit prioritization and completion before starting new work
- Prevents the scope creep that historically pulled Rivi into execution-level work
- Any request to add a fourth concurrent project requires completing or shelving an existing one

## Expiry / Review

Active as long as the Innovation Program operating model remains in effect. Should be reviewed if team size changes significantly.
