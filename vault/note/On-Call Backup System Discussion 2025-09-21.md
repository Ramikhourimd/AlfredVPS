# On-Call Backup System Discussion 2025-09-21

## Context

This is the concluding segment of a multi-person call at [[project/Telia'z Clinic Israel]] on 2025-09-21, following up on the on-call arrangements established in the [[conversation/Rami Oren On-Call Compensation Discussion 2025-09-01]]. Three participants discussed implementing an automated backup system for the on-call doctor roster. [[person/Rami Khouri]] is explicitly named; the other participants are not identified by full name in the recording.

## Proposed Backup System

The participants agreed in principle on an automated backup mechanism for the on-call doctor system:

1. **Backup doctor always designated** — In addition to the primary on-call doctor, a second doctor is always designated as backup (e.g., if the primary is Doctor A, the backup is Doctor B).
2. **Automatic escalation** — If the on-call doctor receives a call and does not acknowledge it (i.e., does not press "1" to confirm within a defined window), the system automatically calls the backup doctor.
3. **Rare emergency use only** — The backup is intended for exceptional situations only (e.g., the primary doctor is sick, in the ER, or had an accident). It is not a routine second line.

## Key Concern Raised

One participant raised a valid concern: the backup doctor will inherently be "less on-call" than the primary, meaning the transition to backup reduces the probability of timely response. The group agreed this is acceptable because:
- The primary on-call doctor is, by definition, in an active readiness state
- The backup is only triggered in genuinely rare emergencies
- The goal is to provide coverage rather than perfection in edge cases

## Definition Clarification

An important definitional point was made: if a doctor is on-call, they are on-call — they cannot simultaneously be in the ER or otherwise unavailable. The backup system addresses force majeure situations, not voluntary unavailability.
