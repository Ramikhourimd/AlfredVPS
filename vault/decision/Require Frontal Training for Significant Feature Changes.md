# Require Frontal Training for Significant Feature Changes

## Context

Psychiatrists were not adopting new features (e.g., in-app summary editing) because they only received WhatsApp updates and PDF training booklets. Some continued using deprecated Google Docs workflows, leading to confusion and support requests.

## Options Considered

1. **WhatsApp announcements only** — cheap but low adoption, psychiatrists don't read long messages
2. **Training booklet distribution** — more detailed but still passive, not confirmed read
3. **Frontal (live) training sessions** — higher effort but confirms understanding and allows Q&A

## Decision

For any significant feature change that alters existing workflows, a live frontal training session must be conducted with the affected professionals before rolling out the change and deprecating the old method.

## Rationale

- Psychiatrists are part-time and do not prioritize reading operational updates
- Sending a booklet does not equal adoption — active demonstration is needed
- The old feature should only be deprecated after confirming all users have transitioned
- Ori Shukron previously conducted effective frontal training that could serve as a model

## Consequences

- Slightly higher coordination overhead for feature rollouts
- Need to schedule sessions with psychiatrists (who have limited availability)
- Alice assigned as liaison between product team and psychiatrists for this process
